Error IY1001: Invalid YAML Syntax

This error occurs when the YAML parser encounters a syntax error in your template file.

COMMON CAUSES:
• Malformed YAML syntax (indentation, missing quotes, etc.)
• Attempting to chain YAML tags directly (e.g., !$map !$if)  
• Missing closing brackets, quotes, or block markers
• Incorrect indentation in block mappings or sequences
• Invalid characters or escape sequences

EXAMPLES:

❌ Incorrect - Chained tags:
  SecurityGroups: !$fromPairs !$map
    source: ["web", "api"]

✅ Correct - Nested structure:
  SecurityGroups: !$fromPairs
    - !$map
      items: ["web", "api"]
      transform: ["{{item}}-sg", { Type: "AWS::EC2::SecurityGroup" }]

❌ Incorrect - Bad indentation:
  Resources:
    S3Bucket:
      Type: "AWS::S3::Bucket"
      Properties:
    BucketName: "my-bucket"  # Wrong indentation

✅ Correct - Proper indentation:
  Resources:
    S3Bucket:
      Type: "AWS::S3::Bucket"
      Properties:
        BucketName: "my-bucket"  # Correct indentation

YAML TAG RULES:
• Only one YAML tag per value (e.g., !$map OR !$if, not both)
• Use nested structures for complex operations
• Check indentation matches YAML block structure
• Ensure all quotes and brackets are properly closed

DEBUGGING TIPS:
• Use a YAML validator to check basic syntax
• Check that line numbers and columns match expected structure  
• Look for missing colons (:) in key-value pairs
• Verify proper spacing after colons and dashes
