Error IY5001: Type Mismatch in Operation

This error occurs when an operation expects one data type but receives another.

COMMON CAUSES:
• Using !$map on a string instead of an array
• Applying array operations to objects
• Mathematical operations on non-numeric values
• String operations on numbers or objects

EXAMPLES:

❌ Incorrect - map on string:
  $defs:
    not_array: "I am a string"
  result: !$map
    items: !$ not_array
    template: "{{item}}"

❌ Incorrect - join on object:
  $defs:
    obj: {key: "value"}
  result: !$join
    array: !$ obj
    delimiter: ","

✅ Correct - map on array:
  $defs:
    my_array: [1, 2, 3]
  result: !$map
    items: !$ my_array
    template: "item {{item}}"

✅ Correct - convert string to array first:
  $defs:
    text: "a,b,c"
  result: !$map
    items: !$split
      string: !$ text
      delimiter: ","
    template: "{{item}}"

TYPE OPERATIONS:
• Arrays: !$map, !$concat, !$join
• Objects: !$merge, !$mapValues
• Strings: !$split, handlebars helpers
• Numbers: arithmetic operations
